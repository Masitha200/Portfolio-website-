System Telemetry & Performance Dashboard

CoreBurner X11 was built to resolve a common problem for system enthusiasts: high-frequency hardware profiling that doesn't tank game frame rates. It aggregates low-level graphics telemetry and processor registers in real time.

The Architectural Challenge

Running high-frequency (5Hz+) system hardware scans typically blocks UI draw calls on single-thread programs. That means telemetry counters lag, causing sensor readings to stutter and skip key heat spikes.

The Engineering Solution

We decoupled sensor telemetry collection from PyQt5 repaint engines. Implementing a background caching thread using non-blocking calls (via psutil and GPUtil) allowed telemetry stats to update at 0.25 seconds without dragging interface frames.
